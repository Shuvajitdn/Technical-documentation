What is C++?

C++ is a general purpose programming language widely used for competitive programming. It includes imperative, object-oriented, and generic programming features and runs on platforms like Windows, Linux, Unix, and Mac.

The “Hello World” program is the first step towards learning any programming language:

#include <iostream> using namespace std; int main() { cout << "Hello World"; return 0; }

The main pillars of Object-Oriented Programming in C++ are:

  • Objects and Classes
  • Inheritance
  • Polymorphism
  • Abstraction
  • Encapsulation
Objects and Classes

Object-oriented programming uses objects to model real-world entities. The goal is to bind together data and functions that operate on that data so they cannot be accessed from outside.

Object: An object is an instance of a class with characteristics and behavior.

Class: A class is a blueprint for creating objects. For example, a 'Car' class can define properties like wheels, speed limits, and mileage.

Inheritance

Inheritance allows a class to acquire properties and methods of another class.

Types of Inheritance:

  • Single Inheritance: Inheriting from one base class.
  • Multiple Inheritance: Inheriting from multiple base classes.
  • Multilevel Inheritance: A derived class created from another derived class.
  • Hierarchical Inheritance: Multiple derived classes from a single base class.
  • Hybrid Inheritance: Combining more than one type of inheritance.
Polymorphism

Polymorphism means having multiple forms. A simple example is a person who is a father, a husband, and an employee.

In C++, polymorphism is divided into:

  • Compile-time Polymorphism (Function or Operator Overloading)
  • Runtime Polymorphism (Function Overriding)
Abstraction

Abstraction is displaying only essential information and hiding the details.

Advantages of Abstraction:

  • Avoids writing low-level code.
  • Prevents code duplication and enhances reusability.
  • Allows changing the internal implementation without affecting the user.
  • Improves security by exposing only important details.
Encapsulation

Encapsulation means wrapping data and the methods that operate on that data into a single unit. It restricts direct access to some of the object's components, which is a means of preventing accidental interference.

In C++, encapsulation is implemented using classes and access specifiers (public, private, and protected).
